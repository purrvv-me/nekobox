NekoBox encrypts every file in your browser before it ever leaves your device. The server only ever stores ciphertext — it never sees your files, your password, or any usable key. It looks and feels like a nostalgic Windows 11 file explorer.

Features

Built like a real product

Not a toy crypto demo — a working file manager with a tested zero-knowledge core.

🧠

Zero-knowledge E2EE

AES-256-GCM with keys derived & used only in the browser. The server stores ciphertext, never plaintext or keys.

🗝️

KEK / VMK hierarchy

A random vault master key wrapped by your password — so changing your password re-wraps one key, not your whole vault.

🔑

Recovery code

A 160-bit code shown once at signup — the only way back in if you forget your password. The server can't help.

🗂️

Folders & drag-drop

Create, rename, move and delete folders. Folder names are encrypted too — the server never learns your structure.

↗️

Secure sharing

Share a file with another user; its key is re-wrapped with their RSA public key, so only they can decrypt it.

📦

Chunked encryption

Large files are encrypted in 4 MiB chunks so they never need to fit in memory all at once.

⌨️

Explorer UX

Search, multi-select, F2/Del/Enter shortcuts, grid & sortable details views, in-browser preview.

🧪

Tested crypto

Vitest covers round-trips, wrong-password rejection, recovery, password change, sharing and tamper detection.

🛡️

Private storage

Cloudflare R2 via short-lived signed URLs, with an encrypted local-disk fallback for development.

How it works

The key hierarchy

Your password unlocks a random vault key. Everything is encrypted under that key — never your password directly.

password ──PBKDF2(SHA-256, 200k, salt)──▶ PWK ──wraps──▶ VMK   (random vault master key)
recovery code ──PBKDF2(…, salt)─────────▶ RWK ──wraps──▶ VMK   (second wrapping)
                                                           │
        ┌───────────────────────────┬───────────────────────┴───────────────┐
        ▼                           ▼                                         ▼
  RSA private key            each file's DEK (AES-256-GCM)            folder & file names
  (for sharing)              wrapped under the VMK                    encrypted under the VMK
                                   │
                            file body encrypted in chunks with the DEK
        sharing → the file's DEK is re-wrapped with the recipient's RSA public key
Educational / portfolio project. The cryptographic design is sound and tested, but it hasn't had a professional security audit. Don't trust real secrets to it without one. Lose both your password and recovery code and your files are gone — by design.
